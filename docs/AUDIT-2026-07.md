### Stack — one correction to CLAUDE.md

Vanilla HTML + CSS, no build step, no framework, Jekyll bypassed via `.nojekyll`
— all confirmed. **But "No JavaScript" is false and always has been.** All 7
games pages carry inline `<script>` blocks: scroll-progress bar (`#scroll-bar`),
hamburger sidebar toggle (`toggleNav()`), a carousel on `games.html`, and a
3D-tilt effect (noted in DESIGN-AUDIT). The canonical games masthead snippet
itself embeds `onclick="toggleNav()"`. The old brief's "No JavaScript **yet**"
was accurate; CLAUDE.md and MEMORY.md's "No JS on the site" hardened it into a
false claim. The real rule the repo follows: **the 8 main-site pages are
JS-free; the games section uses small inline enhancement JS.** Docs should say
that.

Fonts: EB Garamond, Special Elite, Share Tech Mono — loaded via `@import` at the
top of `style.css` and `games-shared.css`, **not** per-page `<link>` tags. The
two imports request slightly different weight sets (style.css includes
Garamond 700; games-shared.css doesn't).

### Tokens

`tokens.css` is genuinely the single source of truth: it holds the only `:root`
block in the repo (canonical 9 + aliases + extras + 6 category tints). Every
page links `tokens.css` → `style.css` (→ `games-shared.css` on games pages), in
that order, with correct `../` prefixes from `now/`. **However**, the tokens.css
header comment still reads "STATUS: created Phase 1. Not yet linked anywhere" —
stale by four phases.

**pre-restructure chrome**:

- all other main pages: `Chicago, IL · 2026` (year present)
- NUL bytes still present on main: `music.html` (20), `games.html` (3),

CLAUDE.md, MEMORY.md, and PROJECT.md all present the locked chrome as
accomplished fact without noting it's unmerged. Screenshots of the live site are
ground truth, and ground truth is the old chrome. **The single highest-leverage
action available is merging (or consciously deciding not to merge)
`feature/canonical-chrome`.** Not done in this audit — flagged for approval.
